# Contributing to Alembic Deploy Action

Thank you for your interest in contributing!

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/alembic-deploy-action.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test your changes
6. Commit: `git commit -m "Description of changes"`
7. Push: `git push origin feature/your-feature-name`
8. Open a Pull Request

## Development Setup

## Testing

Before submitting a PR:

1. Test your changes with the included test suite
2. Ensure all tests pass: Check GitHub Actions
3. Test manually with a real workflow
4. Verify no breaking changes

## Code Style

- Use clear, descriptive variable names
- Add comments for complex logic
- Follow existing code structure
- Keep functions focused and modular
- No emojis in code or logs

## Commit Messages

- Use clear, descriptive commit messages
- Start with a verb (Add, Fix, Update, Remove)
- Reference issue numbers when applicable
- Example: "Fix revision parsing for numeric IDs (#123)"

## Pull Request Process

1. Update README.md with any new features or changes
2. Update examples if workflow syntax changes
3. Ensure all tests pass
4. Request review from maintainers
5. Address review feedback

## Reporting Bugs

Use the bug report template when creating issues. Include:
- Clear description
- Steps to reproduce
- Expected vs actual behavior
- Workflow configuration
- Action output logs

## Feature Requests

Use the feature request template. Include:
- Use case description
- Proposed solution
- Alternatives considered

## Questions?

Open a discussion or issue for questions about contributing.
